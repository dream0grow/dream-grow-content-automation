# S_컨설팅 첨삭 피드백 모음집

곤팀장이 2020년부터 4년간 수백 명의 브랜드블로그 컨설팅·첨삭을 진행하며 노션에 축적한 공통 실수 패턴 모음집. 77페이지 PDF, 38,718자 분량.

## 한 줄 요약

브랜드블로그·콘텐츠 작성에서 수백 명이 똑같이 반복한 실수를 (1) 컨설팅 피드백 3축(제목·표현·논리), (2) 첨삭 피드백 3축(인트로·본문·디자인), (3) Q&A 22선, (4) 발행 직전 체크리스트로 정리한 실전 첨삭 사전.

## 핵심 메시지

- **컨설팅과 첨삭은 다르다.** 컨설팅 = 방향 설정(제목·표현·논리). 첨삭 = 실행 점검(인트로·본문·디자인).
- **논리는 MECE다.** 중복(의미 없이 겹침), 착오(다뤄야 하는데 놓침), 비약(필요한 단계 건너뜀) 3가지가 없는 글이 좋은 논리. *로지컬 씽킹* 인용.
- **3의 법칙.** 글에서는 항상 3가지·3단으로 설명한다. 개수 자체가 빈틈을 막는다.
- **인트로의 90%는 결론부터.** 첫 3~10초 안에 90% 이탈. 두괄식 + 결론 선언 + 근거 3가지가 표준.
- **반복은 다른 말로 변형해서 한다.** "같은 말을 다른 표현으로 서론·본론·결론에 분산"이 세뇌의 기본.
- **글쓰기 전 초안 작성 1단계가 논리를 살린다.** 초안 = 주제 + 각 파트 구분선. 살을 붙일 때 논리가 망가지지 않게 하는 안전장치.
- **비유의 영역을 확장하라.** 자기 업계와 교차하는 4~5개 다른 영역(드라마, 낚시, 스피치, 부동산 등)에서 공통 속성을 끌어와 비유하면 반박 불가능한 논리가 형성된다.

## 컨설팅 피드백 3축

### 1. 제목
- 정직하기만 한 제목은 클릭이 안 된다. 후킹 필수.
- 의미 없는 특수문자는 네이버 어뷰징 규정에 걸린다.
- 100자 제한 안에서 키워드 + 호기심 + 약속을 모두 담는다.

### 2. 표현
- 추상어 대신 구체어. "효율적"보다 "10분에 끝나는".
- 수치·기간·이름을 명시. 일반 명사는 신뢰도를 떨어뜨림.

### 3. 논리 (MECE + 3의 법칙)
1. 주장 1개에 근거 3개
2. 근거 3개는 상호 유기적으로 연결되어야 함 (1번과 3번이 어긋나면 즉시 무너짐)
3. 중복·착오·비약 없음 확인
4. 서론·본론·결론에 같은 메시지를 다른 말로 반복(세뇌)
5. 4~5개 다른 영역에서의 비유로 확장 (다재다능한 인상 + 글감 무한 확장)

## 첨삭 피드백 3축

### 1. 인트로 (첫 3~10초)
- 결론부터. 가벼운 인사로 시작하면 90% 이탈.
- 후킹 3가지 중 1가지: (1) 독자의 기존 생각 파괴, (2) 공포감, (3) 구체적 이득 제시.

### 2. 본문
- 키워드 무지성 3~6번 반복은 옛날 공식, 지금은 자연스러운 맥락 안에 녹여야.
- 체류시간이 핵심 지수.

### 3. 디자인
- 사진·동영상의 적정 비율과 위치
- 글머리 기호·구분선의 의도적 배치

## 발행 직전 체크리스트

(원본 4번 항목)
- 제목 100자 + 후킹 3종 중 1개 사용했는가
- 인트로 결론부터 + 후킹 1개 적용했는가
- 본문 MECE + 3의 법칙 충족했는가
- 비유의 영역 1~2개 추가했는가
- 발행 후 검색해보고 상위노출 확인했는가

## Dream_Grow 활용 가능 영역

- **스레드 작성**: 인트로 = 결론 선언 + 후킹 1개. 본문 = MECE 3축 + 교실 경험. 마무리 = "아이가 ___ 자라길 바랍니다 -Dream_Grow-"
- **부모 교육 글**: 추상어 사용 시 자체 검열 ("효과적"→ "5분 이내", "많은 부모"→ "초등 학부모 30명 중")
- **콘텐츠 첨삭**: 자녀 글이나 학생 글 첨삭 시 (1) 인트로, (2) 본문, (3) 디자인 3축 그대로 적용
- **diff 학습 연계**: 사용자 수정 패턴이 곤팀장 첨삭 패턴과 일치하는지 비교 검증

## 평가

- **장점**: 추상적 글쓰기 조언이 아니라 "이걸 보고 이렇게 고쳐라" 형태의 사전화. 즉시 적용 가능.
- **한계**: 마케팅·세일즈 맥락이 강해 교육·정서 콘텐츠에 그대로 적용 시 과한 자극 발생 가능. 'A가 아니라 B' 논증 구조와 결합해 사용해야 부드러워짐.
- **블로그 외 매체 적용**: 인트로·논리·체류시간 원칙은 스레드·뉴스레터·릴스 모두 호환. 다만 채널 길이에 맞춰 변형 필요.

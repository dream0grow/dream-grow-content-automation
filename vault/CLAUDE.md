# 초생산 볼트 헌법 (v2.1)

이 폴더는 옵시디언 볼트 「초생산」이다. 로컬에서는 이 저장소를 clone한 뒤
`vault/`를 옵시디언 볼트로 열고 Obsidian Git 플러그인으로 동기화한다
(설정 절차: `docs/OBSIDIAN_SETUP.md`).

> 이 파일은 통합기획 v2/v2.1이 정의한 **볼트 헌법**이다.
> 볼트를 읽고 쓰는 모든 세션·에이전트·스크립트는 아래 규칙을 지킨다.

## 볼트에는 md와 첨부만

`.py`, `__pycache__`, `node_modules`, 영상 원본은 볼트 반입 금지.
코드의 진실원천은 이 저장소의 볼트 밖 영역(orchestrator/, vault_pipeline/ 등) 하나다.

## 폴더 구조 — 번호는 생각이 자라는 단계

```
vault/
├── CLAUDE.md              ← 이 파일 (헌법)
├── 수집함/
│   └── plaud/             ← 플라우드 전사 인박스 (API 미인증 시 수동 투입 경로)
├── 제텔카스텐/
│   ├── 0. 시스템/         ← 분류표·태그·템플릿
│   ├── 1. 메모/           ← 원자 메모 (1메모=1아이디어, 원문 발췌는 verbatim)
│   ├── 2. 키워드/         ← K -  (AI 구조화는 K_ai - )
│   ├── 3. 의견/           ← O -  (본인이 표명한 의견만. AI 창작 금지)
│   ├── 4. 주장/           ← P -  ("반론과 한계" 없는 P는 승격 불가)
│   ├── 5. 글감/           ← 주장을 엮은 최종 글감
│   └── 6. 사례은행/
│       ├── _inbox/        ← 형식 없이 던지는 입구
│       └── _노랑대기/      ← 신호등 노랑 판정 사례 (사람 결재 대기)
├── 발행/                   ← 콘텐츠 초안 착륙장. 발행은 반드시 사람이 한다
│   ├── 블로그_교사/        ← 교사 대상 블로그 초안 (복붙 발행)
│   └── 페이스북_교사/      ← 교사 대상 페이스북 초안 (복붙 발행)
└── _system/
    ├── review_queue.md    ← 에이전트→사람 질문·결재함
    ├── lessons.md         ← 실수 학습 루프
    └── logs/              ← 파이프라인 실행 로그
```

기존 초생산 볼트(raw/, wiki/, SNS 콘텐츠 제작 시스템/ 등)는 로컬 이관 스크립트
(`tools/vault_migrate.py`)로 이 구조에 합류한다. 이관 전까지 이 골격은 신규 산출물 전용이다.

## 쓰기 권한 매트릭스 (완화 금지)

| 주체 | 수집함 | 제텔카스텐 1~5 | 6. 사례은행 | 발행/ | _system |
|---|---|---|---|---|---|
| plaud 파이프라인 (자동) | 읽기 | 신규 생성만 | 초록=입고, 노랑=_노랑대기만 | 초안 생성만 | 로그·질문 append |
| 사람 (이한결) | 전부 | 전부 | 전부 | 전부(발행 확정) | 전부 |
| 기타 콘텐츠 에이전트 | 읽기 | **읽기만** | 읽기 | 쓰기 | 질문 append |

- **기존 메모 자동 수정 금지.** 자동화는 새 파일 생성과 `used_in` 이력 추가만 한다.
- `제텔카스텐/1. 개인/`(이관 후 생기면)은 전 에이전트 접근 금지이며 git 커밋도 금지(.gitignore).
- 실패한 AI 처리 결과는 파일로 저장하지 않는다. `_system/logs/`에만 기록.

## 저자 규칙 — `_ai` 딱지와 승격 의식

- frontmatter `author`: 메모의 **문장을 쓴 사람**. `이한결` / `이한결(구술)` / `AI`.
- AI가 작성한 메모는 접두사에 `_ai` (`K_ai - `, `O_ai - `) + `author: AI`.
- 플라우드 녹음에서 **발화를 그대로 발췌**한 메모·의견·사례는 `author: 이한결(구술)`,
  `verbatim: true`. AI는 자르고 나를 뿐 다시 쓰지 않는다.
- **강철 규칙: `_ai` 메모는 4단계(주장)·5단계(글감)에 인용 금지.**
  이한결이 읽고 자기 언어로 고치면 `_ai`를 떼고 author를 이한결로 바꾼다(승격 의식).

## 사례 신호등 (사례은행 입고 판정)

- **초록** = 배움·사고·감정이 드러나고 개인정보 없는 발화 → 가명 처리 후 자동 입고.
  발화 원문은 한 글자도 다듬지 않고 따옴표로 보존, 맥락만 요약 부착.
- **노랑** = 제3의 아이 등장, 갈등 상황, 판단이 애매함 → `_노랑대기/` + review_queue 결재.
- **빨강** = 민감 범주(가정사·건강·경제·종교), 학대·폭력 의심, 조합 시 특정 가능
  → **내용을 어디에도 저장하지 않는다.** 로그에 "차단 N건(사유)" 한 줄만.
- 존엄 테스트: "이 발화가 책에 실려 10년 뒤 성인이 된 그 아이가 읽어도 괜찮은가?"
  확신 없으면 노랑. 학생 실명 금지와 빨강 범주는 학습으로도 완화하지 않는다.

## 발행 안전 규칙

- `발행/` 안 초안의 frontmatter `상태: 리뷰대기`는 자동화가 만들고,
  `발행완료`는 **사람만** 바꾼다. 네이버 블로그·페이스북 게시는 사람이 복사해 발행한다.
- 명시적 확정 원칙: "괜찮네" 같은 모호한 표현은 발행 사유가 아니다.

## 출처 사슬

모든 자동 생성 노트의 frontmatter에 `출처: plaud:<file_id>`(또는 원본 경로)를 기록한다.
이 필드가 중복 처리 방지 로그를 겸한다. 출처 없는 메모는 집필에 인용할 수 없다.

첫 번째: Env-Up

클로드 코드 새 버전 나오면
내 시스템에 영향 있는지 자동으로 체크합니다.

이렇게요.

  1. claude --version으로 현재 버전 확인
  2. GitHub에서 릴리즈 노트 자동 수집
  3. 내가 추적하는 Known Issues 46건 전수 대조
  4. "이거 해결됐네?" → 우회 코드 제거 대상 표시
  5. "이 신기능으로 내 Hook 대체 가능?" → 기능 교체 검토

커맨드 하나 치면 끝.
릴리즈 노트 읽고, 이슈 확인하고,
영향도 판단하고, 문서 갱신까지 전부 자동.

오늘 메인 작업하다 잠시 미뤄뒀었던
v2.1.97~101 네 개 버전 한번에 처리했는데
이슈 4건 해결 확인 + 보안 강화 3건 감지.

두 번째: Kaizen Loop

외부 변화는 Env-Up이 잡고,
내부 품질은 카이젠이 잡습니다.

파이썬 파일 하나 실행하면
AQI 6축 + 인프라 3축 = 9축으로 한번에 측정해요.

  AQI 6축 (에이전트/스킬 품질):
 - 위임 정확도: 맞는 에이전트한테 맡겼나
 - 역할 집중도: 딴짓 안 했나
 - 제약 준수율: 규칙 잘 지켰나
 - 산출물 품질: 결과물 쓸만한가
 - 컨텍스트 효율: 토큰 낭비 없었나
 - 비용 효율: 돈값 했나

  인프라 3축 (운영 건강도):
 - Hook 건강도: 30개 Hook이 에러 없이 도는가
 - 규칙 파일 건강도: 문서가 최신인가, 모순 없나
 - 운영 병목: MCP 느린가, 압축 과한가

오늘 돌렸더니 AQI 81.4점.
Hook에서 에러를 몰래 삼키고 있는 게 72건 잡혔고,
바로 고치라고 시켰습니다.￼

이게 왜 좋냐면,
LLM한테 "너 스스로 평가해봐"라고 시키면
자기가 만든 걸 자기가 채점하는 거라
항상 후하게 나옵니다.

스크립트가 냉정하게 숫자를 매기고
LLM은 그 숫자를 해석하고 고치는 데만 쓰는 거예요.
측정과 판단을 분리한 겁니다.

이러면 나는 LLM이랑 처음부터 지지고 볶을 필요가 없어요.
스크립트가 뽑아준 점수표 보고
"이거 고쳐" 한마디면 끝.
의사결정권자 위치에서 승인만 하면 되는 구조입니다.￼

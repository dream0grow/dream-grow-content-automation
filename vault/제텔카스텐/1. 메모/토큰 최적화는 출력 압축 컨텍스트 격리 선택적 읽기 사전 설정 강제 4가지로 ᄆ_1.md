Claude Code 토큰 최적화 도구 10개를 분류하면 4가지 접근으로 묶인다. (1) 출력 압축은 모델 응답 토큰 자체를 줄이는 방식(Caveman, RTK), (2) 컨텍스트 격리는 원시 출력을 외부 저장소로 빼는 방식(Context Mode, Claude Context), (3) 선택적 읽기는 거대 코드베이스에서 필요 부분만 읽도록 하는 방식(Code Review Graph, Token Savior), (4) 사전 설정 강제는 시스템 프롬프트로 간결성을 강제하는 방식(Token Optimizer, Token Efficient)이다. 작은 프로젝트는 4번만으로 충분하고, 모노레포는 3+2 조합이 효과적이다.

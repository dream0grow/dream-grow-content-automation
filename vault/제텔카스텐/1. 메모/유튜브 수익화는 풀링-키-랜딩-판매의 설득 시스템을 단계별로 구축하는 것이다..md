비즈니스 PT 전체 요약 

왜 유튜브를 하고 있는가? 
수익화 



1주차 = 수익화 구조 기초 : 수익화 구조부터 만든다. 수익화 구조는 설득해서 판매하는 것.  [풀링 - 키 - 랜딩 - 판매]

2주차~4주차 = 기획 

2주차 = 수익화 기획 - 무엇을 팔지 기획하는 것 (무엇을 팔지 정하는 게 제일 중요하다. 상품을 정확히 파악하는 게, 아는 게 필요.)- 사업의 유무에 따라 기획 프로세스가 다름.
3주차 = 키콘텐츠 기획 - 현상 욕구 계획 행동 보상 - 
4주차 = 풀링 콘텐츠 기획 - 풀링 컨텐츠는 사람을 모으는 것 (어떤 사람을 모을 것인가) / 종류 4개 : 에버그린, 시즌성, 데일리, 히어로 / 유튜브 성공 4가지 요소 (들어오게하기(클릭률), 계속 보게 하기(시청지속시간), 다시 보게 하기(구독), 수익화(판매) )

5주차~11주차 = 풀링 콘텐츠 제작

5주차 - 풀링 컨텐츠 제작 기초 (콘텐츠 개괄)
 - 전체 구조 :  기획 (수익화 기획 - 키콘텐츠 기획 - 풀링 콘텐츠 기획) - 제작 (풀링 - 키 - 랜딩) - 판매 (수익화 시스템 - 무료/할인/정가/고가) 
 - 풀링 컨텐츠 시스템 = 콘텐츠 가치 / 개인가치 / 채널 가치  -> 우리가 콘텐츠 가치로 가야하는 이유 
 - 개인 가치 올리는 법 : 캐릭터 롤모델 설정해 분위기 가져오기 / 내적 요인으로 15가지 질문 답해 녹이기 / 콘텐츠 가치를 개인가치로 바꾸기 
 - 채널 가치 올리는 법 : 히트 콘텐츠를 시리즈화 / 관점 정하기 / 결 일치 시키기 
 - 콘텐츠 가치 올리는 법 : 정량적요소 (클릭률과 시청지속시간), 정성적 요소 (정보,공감,재미, 흥미) 

6주차 = 썸네일, 제목 
7주차 = 원고 작성 : 공감 정보 재미 흥미 / 문제, 사례, 유인 제시 
8주차 = 원고 작성 + 촬영 : 정보 = 주장 근거 원리 예시 반론 대안 / 촬영 
		원고 = 공감 - 정보 - 개인가치 - 행동 버튼 - 결론 
9주차 = 촬영, 편집 
10주차 = 업로드 /  구매까지 오는 레벨 1~6 : 현상 인식 - 문제 인식 - 로열티 - 구매욕구 - 구매 경험  
11주차 = 요약

12주차~17주차 = 랜딩 페이지 제작 

12주차 = 랜딩 개괄 
 - 희소성 원칙 : 공급이 수요보다 적어야
 - 구매와 판매 관계 : 
 - 가수요 측정하기 
 - 랜딩페이지의 역할 
  1) 수요측정 -> 공급 통제  (DB쌓기 = 전화번호 수집)
  2) 문구로 가치 부여하기 
 - 랜딩페이지 기본 구조 : 문제가 있다. 문제 잘못해결하면 이렇게 된다. 이렇게 해결하라. 그런데 이런 한계 존재. 이 한계 극복하려면 내 상품으로 대안 
  3) 욕구의 확장
  4) 메시지 보호막 돌파 

13주차 = 무료 상품 만들기 
14주차 = 고정 댓글 + 랜딩페이지 업무 프로세스 (문제 찾기 - 아이템 특징 분석 - 교집합 찾기 - 고정댓글 뽑기 - 차별화 작업 - 이야기 구성 - 랜딩 페이지 양식 채우기) 
15주차 = DB의 가치를 계산하기 
16주차 = 콘텐츠 관리 시트로 수익화 점검
17주차 = 총 정리

18주차 ~23주차 = 키 콘텐츠 제작 
18주차 = 개괄 
 - 키콘텐츠 역할, 목적 : 내 상품의 가치를 높게 여기는 사람들을 모으는 역할
 - 현상, 욕구, 계획, 행동, 보상 
19주차 = 키콘텐츠 제작 실무 
  - 키콘텐츠 기본 구조 : 현상 (후보 지우기)- 욕구(비교군 찾아 분석) -계획 (베스트 아이템 선정) - 행동 (구체적 상품 및 서비스 제시) - 보상 (구매행동에 대한 보상) 
  - 시장 조사 / 현상-욕구-계획-행동 - 보상 + 콤비네이션 
20주차 = 썸네일, 제목디벨롭
  - 제목 : 키워드 후보 찾기 - 판다랭크로 많이 쓰는 표현 선택 - 콘텐츠 격차 확인하기 - 앞에 배치
 21주차 = 키콘텐츠 원고 및 잘팔리는 콘텐츠
  - 도입부(문제,사례,유인) + 본문(주장-근거-원리-사례-반론-대안)+키콘텐츠 핵심 요소(현상-욕구-계획-행동-보상)들 조합해서 원고작성  
  - 잘 팔리는 콘텐츠 6개 : 독점 상품 - 우리꺼만 이용 / 특정 상품 이용시 보다 효과적 / 문제해결 비포 애프터 제공 / 문제해결된 사람 제시 (히어로) / 문제해결해주는 친숙한 제품군과 엮기 / 문제가 없는 사람에게 문제 인식시키기 

22주차 = 키콘텐츠 분석하기 
 - 채널 페이지로 키 콘텐츠 업로드 : 성과 분석시 채널 페이지 유입은 구독 전환 높음
 - 기타 유튜브 기능 : 고정댓글 유입 

23주차 = 요약 

24주차~29주차 = 콘텐츠 확장 (숏폼 + 스레드) 

 24주차 = 개괄
 - 유튜브와 인스타 차이 :인스타가 유행이 빠르다. 인스타는 네트워킹을 위한 플랫폼 - 댓글, 저장, 공유에 게시물의 확산 차이가 남. 
 - 프로필 링크 중요함.
 25주차 = 시장 조사 계정 세팅하기 
  - 잘 되는 것을 수집해서 벤치마킹하자. 
  - 잘 되는 것 수집은 탐색에서 노출된 것보다 더 잘나갔던 콘텐츠 찾아서 벤치마킹
 26주차 = 프로필 세팅 
   - 프로필 세팅도 계정 조사해서 정리하고 잘 되는 것으로 벤치마킹
 27주차 = 숏폼 원고 작성 
   - 주제 디벨롭 (주제를 방향성까지 뽑기) / 구조 디벨롭
   - 표시 제목 -> 원고 작성 : 유인 문제 사례 중 한가지만 / 같은 주제 다른 구조 사용해서 도입부 작성 하기 
  28주차 = 원소스 멀티유즈 (릴스 쇼츠 틱톡 확장) 
    - 표시 제목만 바꾸기 (그 플랫폼에 맞게 구조 넣어서) 
    29주차 = 요약

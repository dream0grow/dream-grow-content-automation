# 초생산 지식 시스템 (Wiki + 제텔카스텐)

이 파일은 초생산 볼트의 지식 관리 규칙을 정의한다.
콘텐츠 생성/발행/리뷰는 Dream_Grow CLAUDE.md가 담당. 이 파일은 **지식 가공과 wiki 관리**에 집중한다.

---

## 구조

```
초생산/
├── CLAUDE.md          ← 이 파일 (지식 시스템 스키마)
├── raw/               ← 원본 소스 (불변, LLM이 수정하지 않음)
│   ├── assets/        ← 이미지, PDF 등 첨부파일
│   └── *.md           ← 웹 클리핑, 논문, 메모, 전사본 등
├── wiki/              ← LLM이 생성·유지하는 위키 (LLM 소유)
│   ├── index.md / log.md
│   ├── sources/ / entities/ / concepts/ / analyses/
├── 제텔카스텐/         ← 지식 가공 시스템
│   ├── 0. 지식창고/    ← 분류표, 태그, 템플릿
│   └── 5. 제텔카스텐/  ← 5단계 파이프라인
├── 투자/              ← 투자 관련 지식 정리
└── SNS 콘텐츠 제작 시스템/ ← Dream_Grow가 관리 (이 세션에서는 읽기만)
```

### 핵심 원칙
- `raw/`는 **불변**. 읽기만, 절대 수정 금지.
- `wiki/`는 **LLM 소유**. LLM이 생성·수정·삭제.
- `제텔카스텐/`은 **LLM이 지식을 가공하여 채운다**.
- `SNS 콘텐츠 제작 시스템/`은 Dream_Grow 관할. 이 세션에서는 읽기만.

본문: 첫 문단 핵심 요약, `[[위키링크]]` 적극 사용, 출처 `[출처:: [[raw/파일명.md]]]`

## 제텔카스텐 5단계 파이프라인

| 단계 | 접두사 | 내용 |
|------|--------|------|
| 1단계 메모 | (없음) | raw에서 추출한 원자적 메모 (1메모=1아이디어) |
| 2단계 키워드 | K - | 핵심개념별, 육하원칙(What/Why/How) 구조화 |
| 3단계 의견 | O - | 동일/반대 의견, 질문 포함 |
| 4단계 주장 | P - | 논증 가능한 주장 (의견들을 엮음) |
| 5단계 두번째 뇌 | (없음) | 주장 조합한 최종 글감 |

키워드 구성: [[핵심개념]] + [[활용목적]] + #태그_트리
스노우볼: 핵심 메모 → 보충 메모 → 사례 순 등급화

## 워크플로우

### 1. Ingest (소스 수집)
사용자가 raw/에 소스 추가 시 **한 번의 처리로 동시 분배**:
1. **제텔카스텐**: 원자적 메모 → `1단계 - 메모/`
2. **wiki/sources**: 소스 요약 (`S_제목.md`)
3. **wiki/concepts**: 새 개념 발견 시 (`C_개념명.md`)
4. **wiki/entities**: 새 인물/조직 발견 시 (`E_이름.md`)
5. **지식창고**: 새 출처 → `B - 출처명.md`, 새 저자 → `W - 저자명.md`
6. **index.md 업데이트** + **log.md 기록**

### 2. Query (질의)
질문 시: index.md → 관련 페이지 읽기 → 종합 답변 (위키링크 명시)

### 3. Zettelkasten 가공
1단계 메모 축적 → 2단계 키워드 페이지 생성/업데이트 → 3단계 의견 → 4단계 주장 → 5단계 글감
Dream_Grow의 `zettel_reader.py`가 4~5단계를 콘텐츠 생성에 활용.

### 4. Lint (월 1회 정비)
1. 페이지 간 모순 확인
2. 고아 페이지 확인
3. 누락 교차 참조
4. 태그 일관성 (분류체계와 일치 여부)
5. 제텔카스텐 ↔ wiki 동기화 (K-파일 ↔ C-파일)
6. 백링크 형식 검증

## 세션 시작

1. 이 CLAUDE.md를 읽는다
2. wiki/index.md를 읽는다
3. wiki/log.md 최근 항목을 읽는다
4. 사용자 요청을 기다린다

## 주의사항

- 소스의 주장을 옮길 때 사실과 의견을 구분한다
- 여러 소스 충돌 시 양쪽 모두 기록, 차이 명시
- raw/ 파일은 절대 수정하지 않는다
- SNS 콘텐츠 제작 시스템/은 이 세션에서 수정하지 않는다 (Dream_Grow 관할)

# 변환 체크리스트

## 스레드 변환 (6~10단락)
- [ ] 핵심 메시지를 한 줄로 압축
- [ ] 1/ 후킹 문장 작성
  - 권위 패턴: "[경력/자격]입니다. [상식 뒤집는 주장]."
  - 충격 패턴: "[놀라운 사실/숫자]. [왜 그런지 알려주겠다]."
  - 공감 패턴: "[타겟의 고민을 구체적으로 묘사]. [해결책 예고]."
- [ ] 2~5/ 본문 핵심 포인트를 단락별로 분리
  - 각 단락 첫 문장이 독립적으로 의미 전달 가능한지 확인
  - 한 단락 = 한 포인트 원칙
- [ ] 마지막/ CTA 작성 (저장/팔로우/댓글 중 하나에 집중)
- [ ] 각 단락이 독립적으로 읽혀도 의미가 통하는지 확인
- [ ] 대화체/관찰체 톤으로 다듬기
- [ ] 해시태그 3~5개 선정

## YouTube 쇼츠 변환 (30~60초)
- [ ] 첫 3초 후킹 문장 작성 (화면 텍스트 + 음성 동시)
- [ ] 핵심 포인트 1~2개로 압축 (쇼츠는 1포인트가 이상적)
- [ ] 타임스탬프 기반 스크립트:
  - 00~03초: 후킹 (자막+음성)
  - 03~10초: 문제/상황 제시
  - 10~25초: 핵심 메시지
  - 25~35초: 증거/사례 (있으면)
  - 마지막 5초: CTA
- [ ] 자막 강조 포인트 표시 (굵게/색상 변경할 부분)
- [ ] 세로 화면(9:16) 기준 구성

## Instagram Reels 변환 (35~60초)
- [ ] 첫 5초 후킹 문장 작성
- [ ] 핵심 포인트 1~3개로 압축
- [ ] 타임스탬프 기반 스크립트:
  - 0~5초: 후킹 (Hook자막 + VO)
  - 5~15초: 문제/공감
  - 15~35초: 핵심 메시지 + 근거
  - 35~45초: 정리 + CTA
- [ ] CTA 작성 (저장/댓글/팔로우)
- [ ] 자막 포인트 표시
- [ ] 캡션 작성 (500자 이내, 핵심 요약 + 해시태그)
- [ ] 커버 이미지 문구 (피드에서 보이는 텍스트)

## 카드뉴스 변환 (5~7장)
- [ ] 1장 표지: 후킹 문구 (큰 글씨, 한 줄)
- [ ] 2장 문제 제시 (타겟의 고민을 구체적으로)
- [ ] 3~5장 핵심 포인트 (한 장 = 한 메시지, 숫자/아이콘 활용)
- [ ] 마지막 장 CTA (저장/공유 유도)
- [ ] 한 장에 한 메시지만 담기
- [ ] 글자 크기: 제목 24pt 이상, 본문 16pt 이상
- [ ] 캡션 작성

## 블로그 변환
- [ ] 제목 SEO 키워드 포함 (네이버/구글 검색량 확인)
- [ ] 영상 스크립트를 읽기 좋은 문장으로 재편집
- [ ] 소제목(H2) 기준으로 구조화 (3~5개 소제목)
- [ ] 핵심 요약 추가 (맨 위 or 맨 아래)
- [ ] 내부 링크 추가 (관련 콘텐츠 연결)
- [ ] CTA 추가 (뉴스레터/채널 구독/상품)

## YouTube 롱폼 변환 (스레드/강의 -> 영상)
- [ ] 유인제시 -> 문제 제시 -> 해결책 구조로 재구성
- [ ] 도입 훅 3개 후보 작성
- [ ] 본문 블록별 근거/사례 보강 (최소 블록당 1사례)
- [ ] 썸네일 문구 후보 3개
- [ ] 제목 후보 3개
- [ ] 시청 유지 장치 배치 ([[C_유머|유머]]/공감/사진전환/질문)
- [ ] 예상 길이: _분
- [ ] 챕터(타임스탬프) 구성

## X 변환
- [ ] 한 줄 주장 (140자 이내)
- [ ] 스레드 버전 (3~5트윗)
- [ ] 인용 유도 문장 ("이것에 대해 어떻게 생각하시나요?")
- [ ] 이미지 첨부 여부

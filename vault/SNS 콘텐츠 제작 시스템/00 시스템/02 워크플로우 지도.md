# 워크플로우 지도

## 현재 실행 중인 흐름

```
흐름 D (스레드)    : AI 생성 → 05 리뷰/대기 → 리뷰/수정 → 05 리뷰/완료 → 발행 → 성과 기록
흐름 E (릴스/숏폼)  : 라이브러리 스레드 기반 → AI 변환 → 05 리뷰/대기 → 리뷰 → 발행 (아직 미실행)
흐름 F (유튜브 롱폼): 원고 작성 → 05 리뷰/대기 → 리뷰 → 촬영 → 편집 → 업로드 (아직 미실행)
흐름 C (뉴스레터)   : 인기 콘텐츠 심화 → 05 리뷰/대기 → 리뷰 → 이메일 발송 (아직 미실행)
```

## 흐름 D. 스레드 생성 (현재 실행 중)

### Step D1. AI가 초안 생성
- AI가 Honcho 메모리에서 채널별 스타일 조회
- 03 라이브러리/38 주제별 콘텐츠/에서 기존 글 참고
- 초안을 05 리뷰/대기/에 저장 (상태: 리뷰대기)
- 원본을 .ai_drafts/에 백업

### Step D2. 사용자 리뷰
폴더: 05 리뷰/대기/
- 옵시디언(모바일/데스크톱)에서 리뷰
- 수정 후 상태를 리뷰완료로 변경
- 발행예정 날짜/시간 입력
- 노트를 05 리뷰/완료/로 이동 (드래그)

### Step D3. 발행
- 발행예정 시간에 Threads에 게시 (수동 또는 자동)
- 상태를 발행완료로 변경

### Step D4. diff 학습 (매일 21시 자동)
- 05 리뷰/완료/ 중 리뷰완료 파일의 수정 패턴 분석
- Honcho에 학습 패턴 저장
- 다음 생성 시 자동 반영

### Step D5. 라이브러리 축적
- 발행완료된 글은 03 라이브러리/38 주제별 콘텐츠/에 복사
- 다음 콘텐츠 생성 시 AI가 참고

## Honcho 메모리 연동

콘텐츠 생성 시 Honcho에서 채널별 스타일 패턴을 조회하여 일관된 톤 유지.
- `peer.chat("스레드 문체 패턴")` → 느낌표/말줄임표/제발 반복/개인경험 패턴
- `peer.chat("릴스 문체 패턴")` → 압축/구어체/인사이트 1개 패턴
- 지원 채널: thread, reels, youtube, blog, newsletter

### diff 학습 (매일 21시 자동)
1. AI가 05 리뷰/대기/에 초안 생성 + .ai_drafts/에 원본 백업
2. 사용자가 옵시디언에서 수정
3. 상태를 리뷰완료로 변경, 05 리뷰/완료/로 이동
4. diff-learn 실행 → Honcho {channel}-corrections에 패턴 저장
5. 발행완료 후 03 라이브러리/38 주제별 콘텐츠/에 복사 (AI 참조용)

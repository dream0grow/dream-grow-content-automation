# plaud-zettel — 플라우드 → 제텔카스텐 → 콘텐츠 파이프라인

**먼저 [`plaud-shared`](../plaud-shared/SKILL.md)를 읽는다** (인증·에러 처리·데이터 모델).
전체 설계 배경: `docs/PLAUD_INTEGRATION.md`

## 목적

플라우드로 녹음한 생각·회의·강의를 그대로 두면 죽은 데이터다. 이 스킬은 녹음을
**원자 노트(제텔카스텐)**로 분해해 지식 베이스를 키우고, 그중 콘텐츠가 될 씨앗을
드림그로우 파이프라인(intake)에 자동 투입한다.

```
플라우드 녹음 → ①수집 → ②원자화 → ③연결 → ④승격(선택) → ⑤처리 기록
```

## 사전 준비

노트는 옵시디언 볼트의 `vault/제텔카스텐/` 폴더에 md 파일로 저장한다(폴더는 이미 존재).
노트 1개 = md 파일 1개. 카드/노트 속성은 파일 frontmatter/본문에 둔다:

- `제목` — 명제형 한 문장 (파일명에도 반영. 예: "아이는 규칙보다 규칙을 만드는 과정을 기억한다")
- `유형` (frontmatter): `임시(fleeting)` / `문헌(literature)` / `영구(permanent)`
- `출처` (frontmatter): `plaud:<file_id>` + 타임스탬프 구간
- `태그` (frontmatter tags): 주제 태그 (예: 훈육, 학습습관, 자존감, 브랜드운영)
- `연결`: 본문에 `[[관련 노트 제목]]` 위키링크로 건다
- `승격` (frontmatter): intake 카드 생성 시 카드 파일 링크 기록

별도 DB 설정·ID 등록은 필요 없다 — 폴더가 곧 저장소이고, GitHub Actions의 git push로 동기화된다.

## 실행 절차

### ① 수집 — 새 녹음 가져오기

- `list_files`로 최근 녹음 조회. 기간을 지정받지 않았으면 **최근 7일**.
- `vault/제텔카스텐/`의 노트 frontmatter `출처`에 이미 있는 `plaud:<file_id>`는
  건너뛴다 (`Grep`으로 검색). 중복 방지의 단일 진실 원천 = 볼트 노트 자체다.
- 녹음마다 `get_note`(AI 요약) 우선, 논지가 불분명할 때만 `get_transcript`.

### ② 원자화 — 녹음 1개 → 노트 N개

- 요약/전사에서 **독립적으로 성립하는 아이디어 단위**로 쪼갠다. 보통 녹음 하나에 1~5개.
- 노트 1개 = 명제형 제목 + 본문 3~7문장(자기 말로 재서술, 전사 복붙 금지) +
  출처(`plaud:<file_id>` + 대략의 타임스탬프).
- 유형 판정: 즉흥 아이디어 → `임시`, 책·강의·타인 발언 요약 → `문헌`,
  기존 노트와 결합해 일반화된 통찰 → `영구`.
- 단순 일정·잡담·실행 지시(할 일)는 노트로 만들지 않는다. 할 일은 사용자에게 목록으로만 보고.

### ③ 연결 — 기존 지식과 잇기

- 노트마다 `Grep`으로 `vault/제텔카스텐/`에서 관련 노트 2~3개를 찾아 본문에 `[[위키링크]]`로 건다.
- 연결하면서 기존 `임시` 노트 2개 이상과 결합해 상위 통찰이 보이면 `영구` 노트를
  새로 만들고 양쪽을 연결한다 (이게 제텔카스텐의 핵심 확장 동작).

### ④ 승격 — 콘텐츠 씨앗 → intake 카드

노트가 아래 기준을 만족하면 **드림그로우 파이프라인 intake 카드**로 승격한다:

- 초등 학부모의 실제 고민/질문에 닿아 있고,
- 드림그로우 브랜드 보이스(경험 기반, 공감형)로 풀 수 있고,
- 기존 파이프라인 카드와 주제가 중복되지 않음 (`vault/파이프라인/` 먼저 검색).

승격 방법: `vault/파이프라인/활성/`에 intake 카드 md 생성 —
frontmatter `stage=intake`, `status=queued`, `format`은 주제 성격에 맞게(thread/newsletter),
`audience`는 기본 "초등 저학년 학부모", 본문에 근거 노트 `[[위키링크]]`와 `plaud:<file_id>` 명시.
이후는 기존 오케스트레이터가 리서치→키워드→초안까지 자동 진행한다.

**승격은 보수적으로**: 확신이 서는 것만. 애매하면 노트로만 남기고 사용자 보고에
"승격 후보"로 표시해 사람이 결정하게 한다.

### ⑤ 처리 기록 + 보고

- 볼트 노트 자체가 처리 로그다 (①의 중복 방지 참조).
- 사용자에게 표로 보고: 처리한 녹음 수 / 생성 노트(제목·유형) / 승격 카드 / 승격 후보 /
  건너뛴 녹음(사유). 발견한 할 일이 있으면 별도 목록.

## 제약·주의

- **인증**: 플라우드 MCP OAuth는 브라우저 콜백이 필요해 원격(웹) 세션에서는 실패할 수 있다.
  `login` 실패 시 사용자에게 두 가지를 안내: ① 로컬 Claude Code에서 이 스킬 실행,
  ② claude.ai 설정 → 커넥터에 `https://mcp.plaud.ai/mcp` 추가 (docs/PLAUD_INTEGRATION.md 참고).
- **GitHub Actions 자동화 불가(현재)**: 오케스트레이터 cron에는 플라우드 인증 수단이 없다.
  자동화 확장은 docs/PLAUD_INTEGRATION.md의 "다음 단계" 참고.
- 제3자 노출 문구를 생성하는 단계가 아니므로 윤문 스킬은 불필요. 단, 승격 카드에
  들어가는 사용자향 문구를 새로 쓰면 CLAUDE.md 윤문 규칙을 따른다.

## 3. 확정된 설계 결정

### 3-1. 구조: 새 폴더로 이사하지 않는다

기존 5단계 파이프라인(1단계 메모 → 2단계 키워드 K → 3단계 의견 O → 4단계 주장 P →
5단계 두번째 뇌)을 그대로 척추로 쓴다. 기존 CLAUDE.md의 공식(1메모=1아이디어,
K 육하원칙, 스노우볼, raw 불변)을 계승한다.

추가 폴더 2개만 신설:
```
제텔카스텐/6. 사례은행/
├── _inbox/          # 형식 없이 던지는 입구 (아이폰 옵시디언에서 한 줄도 OK)
└── (규격화된 사례 카드들)
_system/
├── values.md        # 글쓰기 가치 기준 (4-3 참조)
├── lessons.md       # 실수 학습 루프
├── review_queue.md  # 에이전트→사람 질문함
└── logs/
```

### 3-2. 메타데이터 스키마 (모든 신규/수정 메모에 적용)

```yaml
---
title: 복지 정책이 나라 경제 성장의 밑거름이다
author: 이한결            # 이 메모의 "문장"을 쓴 사람: 이한결 | AI
source_type: paper        # book|paper|youtube|hwp|own_script|own_content|experience
source_author:            # 원본을 만든 사람: 책 저자, 논문 저자, 유튜브 크리에이터 등
source_file: raw/Roam-Export/영구메모.md
source_original: 낙수효과의 개념과 실제에 관한 연구.pdf
source_section: 16~18쪽
verbatim: true            # 원문 그대로 옮긴 경우 true (AI 재작성 없음)
created: 2026-07-04
links: []
---
```

핵심 구분: **author = 메모를 쓴 사람 / source_author = 지식의 원천인 사람.**
예: 이한결이 후버만 영상을 소화해 쓴 원고에서 나온 메모는
author: 이한결, source_author: 앤드류 후버만.
→ 집필 시 author가 이한결인 문장은 본문 목소리, source_author가 타인인 지식은
"~에 따르면" 인용 목소리로 자동 구분한다.

source_type별 규칙:
- own_content(자기 SNS 콘텐츠가 출처): "원출처 추적 필요" 플래그 자동 부착
- experience(교실 관찰 등): 책에는 사용 가능. 논문에 사용 시 "[학술 근거 보강 필요]" 자동 표시

### 3-3. 제목 규칙: AI 딱지와 승격 의식

- AI가 작성한 메모: 접두사에 `_ai` → `K_ai - `, `O_ai - `, `P_ai - ` (+ frontmatter author: AI)
- 이한결이 쓰거나 승격한 메모: 깨끗한 `K - `, `O - `, `P - `
- **승격 의식**: 이한결이 `_ai` 메모를 읽고 자기 언어로 고치면 `_ai`를 떼고 author를 이한결로 변경
- **강철 규칙: `_ai` 딱지 메모는 4단계(주장)와 5단계(글감)에 인용 금지.**
  AI 메모는 재료, 이한결의 눈을 통과한 것만 벽돌이다.
- 기존 제텔카스텐 메모 약 2,000개는 전량 소급 `_ai` 마킹 (Phase 1 스크립트)

### 3-4. 사례 규칙 (개정판)

> **AI는 사례를 창작하지 않는다. 이한결이 쓴 사례는 언제 어디서 들어와도 환영하며,
> 시스템은 그것을 받아 규격화하는 일만 한다.**

- 원고에 사례가 필요한 자리: 창작 대신 빈 카드만 남긴다.
  예: `[사례 요청: 혼내도 같은 행동을 반복하던 아이의 실제 관찰. 학년·상황·1년 뒤 변화가 있으면 좋음]`
- 사례 공급 경로 (열려 있음, 추가 가능):
  1. `6. 사례은행/_inbox/`에 아무 형식으로 던지기 (밤에 에이전트가 규격화)
  2. 원고의 [사례 요청] 카드에 직접 채우기 (아침 루틴)
  3. 에이전트가 일간노트·Roam에서 과거 기록을 원문 발췌해 후보 제시
  4. **Plaud 음성 파이프라인 (6장 참조)**
- 규격화 시 에이전트 권한: frontmatter 부착 + 오탈자 교정 + 키워드 연결까지만.
  문장 각색·살 붙이기 금지 (붙이는 순간 AI 사례가 됨).
- 검수 에이전트: 최종 원고의 모든 사례 문장에 source_type: experience 출처가
  있는지 전수 검사, 없으면 반려. 기존 챕터(예: 챕터02 "복도에서 뛰는 아이")도
  소급 검사 — 실화면 출처 부착, 아니면 [사례 요청] 카드로 교체.

### 3-5. 보안 규칙 (Phase 0, 최우선)

- `제텔카스텐/1. 개인/0.개인정보/` 폴더와 `인박스_메모/개인통관번호.md` 등
  비밀 정보를 볼트 밖(패스워드 매니저)으로 반출
- **API 키는 재발급(로테이션)** — zip으로 외부 업로드된 이력이 있음
- git 히스토리에서 해당 파일 제거 (git filter-repo 등)
- 모든 에이전트 공통 규칙: `1. 개인/` 폴더 접근 금지
- 실패한 AI 처리 결과는 파일로 저장하지 않고 로그에만 기록 (_검토필요 오염 재발 방지)

## 4. 10개 사고 공식의 파이프라인 배치

공식의 수식을 계산하지 않는다. 각 공식의 사고 구조를 단계별 규칙으로 번역한다.

| 단계 | 공식 | 번역된 규칙 |
|------|------|------------|
| 1단계 메모 | GI의 O(관찰) | "옮겨 심기": 요약·해석 없이 원문 보존 (verbatim: true) |
| 2단계 키워드 | MDA 5차원 | K파일에 5차원 질문: 시간(언제 왜 등장, 지금도 유효한가)·공간(제주/한국/세계에서 어떻게 다른가)·추상(일반화하면 어떤 원리인가)·인과(어떤 경로로 작동하는가)·계층(교실/학교/제도에서 각각 무엇인가) |
| 2단계 연결 | CC 매트릭스 | 링크 생성 시 3줄 의무: 공통점(∩) / 차이점(⊕) / 전이 f(A→B) "A의 통찰을 B에 옮기면?" |
| 3단계 의견 | PR 재정의 | 토론 에이전트의 새벽 3질문: ①통념을 180° 뒤집으면? ②교실 한 칸(0.1x)과 제도 전체(10x)에서는? ③이것은 사실 무엇에 관한 문제인가(ψ+1)? → 이한결의 "A가 아니라 B" 문형과 동일 구조 |
| 3→4 증폭 | IA | 소크라테스 후속 질문 3유형 고정: Why×5 / What if / How might we. C(협력)와 Q(질문의 질)가 곱셈 항 = 토론 에이전트의 존재 이유 |
| 4단계 주장 | GI의 분모(고정관념+편향) | **"반론과 한계" 섹션 없는 P노트는 승격 불가** (검수 에이전트가 관문) |
| 5단계 글감 | IS + CS | 조합·타분야 차용·제약 활용·역방향 / 자산이 가장 두터운 챕터부터 (레버리지 포인트) |
| 운영 | TE | lessons.md 누적 루프 (학습+경험+반성의 적분) |
| 운영 | IL 직관 | **AI가 대신 못 하는 유일한 공식.** 아침 루틴: 논리로 답하기 전에 첫 직관을 판단 없이 한 줄 먼저 기록. 에이전트는 직관 줄에 토 달기 금지 |
| 운영 | IW | 검수 채점표에 겸손(H)·윤리(E)·공감(C) 항목. 지식×실행이 커도 겸손·윤리가 0이면 지혜는 0 |

## 5. 에이전트 정의 (`.claude/agents/`에 생성)

공통 규칙 (볼트 CLAUDE.md에 추가):
- raw/ 읽기 전용, 1. 개인/ 접근 금지
- 실패 시 파일 생성 금지, 로그만
- 확신 60% 미만이면 review_queue.md에 질문 후 건너뜀
- 사례·의견·주장 창작 금지
- 모든 산출물에 3-2 스키마 적용

### ① transplanter (이식 에이전트) — Phase 2의 주역
Roam 원문을 "자르되 다시 쓰지 않는다". 최상위 불릿 1개 = 1단계 메모 1개.
author: 이한결, verbatim: true. 원문의 `출처::`, 논문 쪽수, [[링크]]를 frontmatter로 이관.
우선순위: 영구메모.md → 유튜브 원고 → 독서노트.md → 일간노트(사례은행행).

### ② converter (파일 변환) — hwp → raw/ (기존 계획 유지)

### ③ classifier (분류) — 1→2단계 + 3대 청소
출처 형식 통일 소급 / K 중복 병합(예: 개념기반탐구 vs 개념기반탐구수업) /
_검토필요 613개 삼분류(AI 오류→삭제, 개인정보→반출 목록, 진짜 메모→1단계 재투입).
링크 생성 시 CC 3줄(공통·차이·전이) 의무.

### ④ socrates (토론) — 2→3단계 병목 전담
매일 새벽 K파일 1개 선정 → PR 3질문 + IA 후속질문 작성 →
_system/dialogues/날짜.md. 이한결의 아침 답변을 그날 밤 O노트로 규격화
(author: 이한결 — 답의 문장이 이한결 것이므로). 직관 줄에는 절대 개입 금지.

### ⑤ writer (집필) — 4→5단계→챕터
스노우볼(핵심→보충→사례) 공식. `_ai` 메모 인용 금지. 모든 주장 옆 [[메모ID]].
사례 자리는 [사례 요청] 카드. 자산이 가장 두터운 챕터의 다음 소제목부터.

### ⑥ researcher (리서치)
1단계 메모의 "추가로 조사할 내용" 필드 주간 수집 → 논문 검색 → raw/papers/ 채우기
(제목·저자·연도·DOI·URL, 가능하면 BibTeX). 순환참조 42건의 원출처 역추적.
experience 출처만 있는 주장의 학술 근거 보강.

### ⑦ reviewer (검수) — 최종 관문
humanize-korean 규칙(AI 문체 제거) + values.md 채점 + 출처 사슬 전수 검사
+ 사례의 experience 출처 검사 + 학생 가명 처리 검사 + "반론과 한계" 유무 검사.

### ⑧ case-keeper (사례 정리) — 신설
_inbox와 Plaud 전사본에서 사례 규격화. 각색 금지, 오탈자 교정과 frontmatter까지만.
학생 실명 자동 가명화("O이", "한 아이"). 주 1회 "교실 언어 리포트"
(반복 발화 패턴 통계 → socrates에게 O노트 후보로 전달).

## 6. Plaud 음성 파이프라인 (설계 확정, 연동 검증 완료)

Plaud 공식 MCP 서버가 존재하며 Claude Code에서 작동 확인됨 (2026년 기준).
연결: Claude 커넥터 목록에서 Plaud 연결(웹/데스크톱/Code 공유) 또는
`https://mcp.plaud.ai/mcp` 커스텀 커넥터. CLI도 제공됨.

전제: 녹음·전사는 교육기록용으로 정리되어 있음 (이한결 확인, 2026-07-04).
아이들의 생생한 발화 원문은 가치가 크므로 **따옴표 원문 그대로 볼트 반입을 허용**한다.
단, 반입 여부는 아래 신호등 판정을 거친다.

흐름:
1. 수업 중 Plaud 녹음 → Plaud 클라우드가 전사·요약
2. 밤: case-keeper가 MCP로 당일 전사본 조회 → 발화·사례 후보 추출 → 신호등 판정
3. 초록은 자동 입고, 노랑은 review_queue, 빨강은 로그만
4. 아침: 이한결이 노랑 결재 → 사례은행 입고 (author: 이한결(구술), source_type: experience)
5. 주 1회: 교실 언어 리포트 (반복·강조·실패 패턴) → socrates의 질문 재료

**발화 신호등 (case-keeper 판정 규칙):**
- 초록(자동 입고): 배움·사고·감정이 드러나고 개인정보 없는 발화.
  가명 처리 후 발화 원문은 한 글자도 다듬지 않고 따옴표로 보존, 맥락만 요약 부착.
- 노랑(검토 대기): 제3의 아이 등장, 갈등 상황, 판단이 애매한 발화.
  가명화 초안 + 사유를 달아 review_queue로. 이한결이 승인/반려.
- 빨강(반입 금지): 가정사·건강·경제 형편·종교 등 민감 범주, 학대·폭력 의심,
  조합 시 아이가 특정되는 발화. review_queue에도 내용을 올리지 않고
  "차단 N건(사유)" 로그 한 줄만 남긴다.

**판정 체크리스트 5:**
① 식별 가능성(가명을 써도 조합으로 특정되면 일반화 또는 노랑)
② 민감 범주 여부(해당 시 빨강)
③ 제3자 등장(해당 아이도 가명화 + 노랑)
④ 교육적 가치(사고·배움을 보여주는가)
⑤ **존엄 테스트: "이 발화가 책에 실려 10년 뒤 성인이 된 그 아이가 읽어도 괜찮은가?"
   확신 없으면 노랑.** (values.md에도 등재)

**학습 루프:** 이한결의 노랑 결재(승인/반려 + 사유)를 lessons.md에 누적 →
case-keeper가 결재 패턴을 학습해 초록 범위를 점진 확대.
고정 규칙: 학생 실명 금지(가명 유지)와 빨강 범주는 학습으로도 완화하지 않는다.

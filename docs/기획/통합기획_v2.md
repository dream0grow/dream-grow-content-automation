## PART 1. 단일 볼트 판정 — "목적에 부합하는가"에 대한 답

### 결론: 부합한다. 단, 세 가지 조건이 지켜질 때만이다.

**단일 볼트가 옳은 이유 4가지:**

1. **플라이휠의 물리적 거리가 0이 된다.** 콘텐츠→지식 환류(candidate)와 지식→콘텐츠 공급(zettel_reader)이 같은 볼트 안 폴더 이동으로 끝난다. 볼트가 둘이면 모든 교차 지점에 경로 설정·권한·동기화 타이밍 문제가 생긴다.
2. **링크 그래프가 하나가 된다.** "훈육" 주장 메모에서 그 주장으로 만든 스레드 글까지 [[위키링크]]로 이어지고, 옵시디언 그래프에서 지식→콘텐츠 계보가 한눈에 보인다. 책 집필 시 "이 주장은 이미 스레드에서 반응 검증됨"을 즉시 확인할 수 있다.
3. **동기화·모바일이 1회로 해결된다.** Obsidian Sync 하나로 지식과 콘텐츠 모두 아이폰에서 열린다.
4. **이미 그렇게 운영 중이고, 두 CLAUDE.md의 관할 분리도 이미 작동한다.** 실물을 계획에 맞추는 것보다 계획을 실물에 맞추는 편이 싸다.

**단일 볼트의 위험과 그 방어 (조건 3가지):**

| 위험 | 방어 조건 |
|------|----------|
| 경계 붕괴: 콘텐츠 에이전트가 지식 폴더를 오염 | **조건 1 — 쓰기 권한 매트릭스** (PART 2-3)를 볼트 루트 CLAUDE.md에 헌법으로 명시. 위반은 reviewer가 주간 감사 |
| 동기화 충돌: Sync + git + (코드의) Dropbox 삼중 | **조건 2 — 역할 단일화**: 동기화=Obsidian Sync 하나. git은 "서버에서만 커밋·푸시하는 단방향 백업 + 에이전트 파괴적 작업 전 안전망"으로 강등, 어떤 기기에서도 pull/merge 금지. 코드는 볼트·Dropbox 밖으로 (PART 3) |
| 볼트 비대화: 코드·캐시·대용량이 Sync를 압박 | **조건 3 — 볼트에는 md와 첨부만.** .py, __pycache__, node_modules, 영상 원본은 볼트 반입 금지 |

### 4-1. 기준 ① 폴더 구조·코드 베이스 청결 — 현재 미달, 처방 완료

### 4-3. 기준 ③ 에이전트별 모델 차등 — 하드코딩 6종 발견, config 일원화로 해결

    "creative_draft":  "claude-fable-5",      # 창작 초안·socrates·writer

### 4-4. "큰 작업을 위한 파일 준비 상태" 판정

사용자의 추정("구축 전이라 없는 것 아닌가")이 정확하다. 준비도 체크리스트:

| 준비물 | 상태 | 비고 |
|--------|------|------|
| Roam 원본(진짜 지식) | ✅ 있음 | 영구메모·유튜브원고 1.28MB·독서노트·일간노트 340개 |
| 콘텐츠 자산·Honcho·발행 파이프라인 | ✅ 가동 중 | 보존 자산 목록 유효 |
| raw/papers (논문 근거) | ❌ 비어 있음 | researcher 임무 — 책 집필 전 필수 |
| _system/ (values, lessons, candidates, review_queue) | ❌ 없음 | Phase 0.5에서 생성 |
| 사례은행 | ❌ 없음 | Phase 0.5에서 생성 |
| `_ai` 마킹·출처 표준화 | ❌ 0건 | Phase 1 |
| 텔레그램 봇·vault_watcher·knowledge_feedback | ❌ 없음 | Phase B·G |
| code_health_rubric·weekly_cleanup | ❌ 없음 | Phase F |
| 모델 config 일원화 | ❌ 하드코딩 6종 | Phase A |
| 보안 (키 관리) | ❌ **2회 유출** | Phase 0 즉시 |

결론: **기초 자산(원료·발행망)은 갖춰졌고, 관제 계층(_system·게이트웨이·청소 루프·모델 정책)이 전무하다.** 지금이 구축 적기라는 뜻이기도 하다 — 관제 계층이 없는 상태에서 자동화를 더 늘리면 _검토필요 613개 같은 오염이 콘텐츠 쪽에서도 재현된다.

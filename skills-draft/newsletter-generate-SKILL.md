## 입력
- 주제 키워드 (필수)
- 카테고리 (필수): 훈육/수학/독서/미디어/놀이/감정/학습/학교/크리에이터
- 시리즈 번호 (선택): 이전 뉴스레터 연결 시
- 참고 자료 경로 (선택): wiki, 제텔카스텐, 기존 스레드/블로그

## 절차

1. **Honcho 컨텍스트 조회**
   ```python
   from content_automation.memory_manager import get_honcho_client, get_full_context
   client = get_honcho_client()
   ctx = get_full_context(client, "newsletter")
   ```
   - style, corrections, brand, team_learnings 모두 참조

2. **참고 자료 수집**
   - wiki/에서 관련 이론/연구 검색
   - 제텔카스텐/에서 연결 메모 탐색
   - 03 라이브러리/38 주제별 콘텐츠/에서 해당 카테고리 기존 콘텐츠 확인
   - 07 스레드/[카테고리]/에서 관련 스레드 참조 (심화 확장 소재)

3. **뉴스레터 초안 작성** (규칙: .claude/rules/10-content-generation.md)
   - **모델: Opus 4.6 사용** (글쓰기 품질 최대화)
   - 6000~7000자, 시리즈 형태 심화 콘텐츠
   - 구조:
     1. **인트로** — 이전 뉴스레터 연결 (시리즈 시) 또는 독자 인사 + 주제 도입
     2. **본론** — 교실 에피소드 + 단계별 방법 (3~5단계)
     3. **[가정에서 연습하는 법]** — 부모-자녀 대화 예시 (구체적 상황 + 대사)
     4. **유의할 점** — 흔한 실수 2~3가지 + 대안
     5. **정리** — 핵심 메시지 요약
     6. **다음 뉴스레터 예고** — 연결 주제 힌트
   - 그로우써클 커뮤니티 명칭 사용
   - %name% 개인화 변수 삽입 (인트로, 마무리)
   - 이모지 금지, 가짜 통계 금지, 'A가 아니라 B이다' 논증
   - 교실 에피소드 최소 1개
   - 마무리: 주제에 맞는 기원문 + "아이와 부모의 꿈을 키웁니다. -Dream_Grow-"

4. **파일 저장**
   - 경로: `05 리뷰/대기/원고_뉴스레터_[카테고리]_[키워드+키워드].md`
   - frontmatter: 주제, 카테고리, 채널: newsletter, 상태: 리뷰대기, 생성일, 출처: AI생성_Dream_Grow스타일
   - 시리즈 정보가 있으면 시리즈/시리즈번호 필드 추가
   - `.ai_drafts/`에 원본 백업 (diff 학습용)

5. **검증**
   - 6000자 이상 확인
   - 이모지 없음 확인
   - % 수치 있으면 출처 확인
   - 부모-자녀 대화 예시 포함 확인
   - 마무리 문구 + Dream_Grow 서명 확인
   - 파일명 규칙 준수 확인
   - %name% 변수 최소 1회 삽입 확인

## 완료 조건
- 05 리뷰/대기/에 뉴스레터 초안 파일 1개 생성
- .ai_drafts/에 백업 1개 생성
- 콘텐츠 규칙 위반 0건
- 6000~7000자 범위 충족
